API Management

Welcome to apiman, open source API management. Use this software to provide various APIs (Services) to your users (Applications) through a secure, scalable, and governed Gateway layer. API Management allows Service Developers to centralize control and analysis of their Services. It also provides a central location for Application Developers to find and consume available APIs.
Organizations
All services and applications must be managed within the context of an Organization. You can be a member of multiple Organizations at the same time, with different roles in each: you can be an Application Developer in one organization and a Service Developer in another.
  • Create a New Organization
  • Browse/Find an Organization
  • Go to My Organizations
Services
Create, find, or manage Services. A Service is also known as an API - anything that can be invoked remotely by some sort of client (Application). This platform simply provides a way to expose existing APIs by defining them as Services and attaching value-added policies to them.
  • Find/Consume a Service
  • Manage My Services
  • Create a New Service
Applications
Create and manage your Applications. An Application is the thing that consumes APIs (Services). These consumers must be defined in this platform so that contracts can be created between them and the Services they wish to consume.
  • Manage My Applications
  • Create a New Application
